# 숫자 야구 게임
## 진행 방법
* 숫자 야구 게임 요구사항을 파악한다.
* 요구사항에 대한 구현을 완료한 후 자신의 github 아이디에 해당하는 브랜치에 Pull Request(이하 PR)를 통해 과제를 제출한다.

## 과제 제출 과정
* [과제 제출 방법](https://github.com/next-step/nextstep-docs/tree/master/precourse)

## 요구사항
* 기본적으로 1부터 9까지 서로 다른 수로 이루어진 3자리의 수를 맞추는 게임이다.
* 같은 수가 같은 자리에 있으면 스트라이크, 다른 자리에 있으면 볼, 같은 수가 전혀 없으면 포볼 또는 낫싱이란 힌트를
얻고, 그 힌트를 이용해서 먼저 상대방(컴퓨터)의 수를 맞추면 승리한다.
  * [예] 상대방(컴퓨터)의 수가 425일 때, 123을 제시한 경우 : 1스트라이크, 456을 제시한 경우 : 1스트라이크 1볼,
  789를 제시한 경우 : 낫싱
* 위 숫자 야구게임에서 상대방의 역할을 컴퓨터가 한다. 컴퓨터는 1에서 9까지 서로 다른 임의의 수 3개를 선택한다. 게
  임 플레이어는 컴퓨터가 생각하고 있는 3개의 숫자를 입력하고, 컴퓨터는 입력한 숫자에 대한 결과를 출력한다.
* 이 같은 과정을 반복해 컴퓨터가 선택한 3개의 숫자를 모두 맞히면 게임이 종료된다.
* 게임을 종료한 후 게임을 다시 시작하거나 완전히 종료할 수 있다.
* 사용자가 잘못된 값을 입력할 경우 [ERROR]로 시작하는 에러 메시지를 출력하고 게임을 계속 진행할 수 있어야 한다.

## 기능 구현 목록
* 상대방(=컴퓨터)
  * 1 ~ 9 까지의 임의의 수 3개를 생성
  * 생성된 세자리 숫자는 사용자가 정답을 맞출때 까지 유지해야 한다.
    * 임의로 생성하는 3개의 수는 서로 다른 숫자이어야 한다.
   <br><br>
* 사용자
  * 1 ~ 9 까지의 임의의 수 3개를 입력
  * 사용자가 입력하는 3개의 수는 서로 다른 숫자이어야 한다.
  <br><br>
* 사용자 입력 검증
  * 숫자 형식 검증(숫자 외에 다른 값이 입력되었는지 확인)
  * 중복 입력 검증(중복된 숫자가 입력되었는지 확인)
  * 사용자가 잘못된 값을 입력할 경우 [ERROR]로 시작하는 에러 메시지를 출력
  <br><br>
* 게임 결과 계산
  * 상대방(=컴퓨터)가 생성한 세자리 수와 사용자가 입력한 세자리 수를 비교
    * 숫자의 값과 위치가 같을 경우 *스트라이크*
    * 숫자의 값이 같지만 위치가 다를 경우 *볼*
    * 숫자의 값이 모두 다를 경우 *낫싱*
    * 게임의 결과 계산 값은 사용자가 새로운 값을 입력할 때마다 초기화 되어야 한다.
  <br><br>
* 사용자가 정답을 맞춘 경우
  * 게임종료 혹은 다시 시작 선택
  * 다시 시작을 선택 했을 경우 상대방(=컴퓨터)는 새로운 임의의 수를 생성
