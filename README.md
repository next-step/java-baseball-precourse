# 숫자 야구 게임
## 진행 방법
* 숫자 야구 게임 요구사항을 파악한다.
* 요구사항에 대한 구현을 완료한 후 자신의 github 아이디에 해당하는 브랜치에 Pull Request(이하 PR)를 통해 과제를 제출한다.

## 과제 제출 과정
* [과제 제출 방법](https://github.com/next-step/nextstep-docs/tree/master/precourse)

## 기능목록
### 컴퓨터의 수를 생성 (=게임시작)
- 1~9사이의 서로다른 랜덤한 3자리수 생성
### 컴퓨터와 플레이어의의 각자리 수를 비교
- 같은 위치 같은 숫자 = 스트라이크
- 다른 위치 같은 숫자 = 볼
### 비교 결과를 생성
- 스크라이크 & 볼의 개수 출력 (단, 0개인 경우 출력하지 않음)
- 스트라이크와 볼의 개수가 모두 0인 경우 낫싱 출력
- 스트라이크가 3개인 경우 정답및 게임종료메세지 출력
### 플레이어의 수를 입력
- 1~9사이의 서로다른 랜덤한 3자리수 입력받음
- 잘못된 입력인 경우 에러메세지 출력후 재입력 받음
### 다시시작 또는 게임종료 판단
- 정답인 경우 다시시작(1) 또는 게임종료(2) 여부를 입력받음
- 유효하지 않은 입력인 경우 에러메세지 출력후 재입력 받음
- 1 입력시 = 게임 다시시작
- 2 입력시 = 게임 종료
