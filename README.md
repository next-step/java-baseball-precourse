# 숫자 야구 게임
## 진행 방법
* 숫자 야구 게임 요구사항을 파악한다.
* 요구사항에 대한 구현을 완료한 후 자신의 github 아이디에 해당하는 브랜치에 Pull Request(이하 PR)를 통해 과제를 제출한다.

## 과제 제출 과정
* [과제 제출 방법](https://github.com/next-step/nextstep-docs/tree/master/precourse)

## 기능 요구사항
- 기본적으로 1부터 9까지 서로 다른 수로 이루어진 3자리의 수를 맞추는 게임이다.
- 같은 수가 같은 자리에 있으면 스트라이크, 다른 자리에 있으면 볼, 같은 수가 전혀 없으면 포볼 또는 낫싱이란 힌트를 얻고, 그 힌트를 이용해서 먼저 상대방(컴퓨터)의 수를 맞추면 승리한다.
- [예] 상대방(컴퓨터)의 수가 425일 때, 123을 제시한 경우 : 1스트라이크, 456을 제시한 경우 : 1스트라이크 1볼, 789를 제시한 경우 : 낫싱
- 위 숫자 야구게임에서 상대방의 역할을 컴퓨터가 한다. 컴퓨터는 1에서 9까지 서로 다른 임의의 수 3개를 선택한다. 게임 플레이어는 컴퓨터가 생각하고 있는 3개의 숫자를 입력하고, 컴퓨터는 입력한 숫자에 대한 결과를 출력한다.
- 이 같은 과정을 반복해 컴퓨터가 선택한 3개의 숫자를 모두 맞히면 게임이 종료된다.
- 게임을 종료한 후 게임을 다시 시작하거나 완전히 종료할 수 있다.
- 사용자가 잘못된 값을 입력할 경우 [ERROR]로 시작하는 에러 메시지를 출력하고 게임을 계속 진행할 수 있어야 한다.

## 기능 목록
### 숫자야구 서비스
- 서로 다른 랜덤한 3자리 수를 생성한다.
  - 자리 수는 추후 수정할 경우 유연하게 가져가기 위해 상수로 선언한다.
  - 랜덤 수 생성 유틸을 사용하여 생성한다.
    - 중복을 체크한다. 중복 발생 시, 에러를 발생하지 않고 다시 랜덤 수를 생성한다.
- 스트라이크 수를 반환한다.
- 볼 수를 반환한다.
- 반환받은 스트라이크와 볼 수를 메시지 처리한다.
  - 스트라이크 또는 볼 수가 0보다 클 경우, 메시지를 처리한다.
  - 같은 수가 전혀 없으면 낫싱 처리한다.

### 스코어 객체
- 스트라이크 갯수 필드를 가진다.
- 볼 갯수 필드를 가진다.

### 게임 컨트롤러
- 게임 시작, 종료, 재시작 분기를 결정할 수 있다.
  - 게임 시작: 숫자야구 컴퓨터를 초기화한다.
  - 게임 종료: 프로그램을 종료한다.
  - 재시작: 새로 시작, 종료 에 따라 게임 시작, 게임 종료를 호출한다.
- 사용자 입력값을 받는다.
  - 입력값을 검증한다.
    - 게임 진행 중: 3개의 서로 다른 숫자를 입력했는지 검사한다.
    - 게임 종료 후: 새로 시작 1, 종료 2만 입력했는지 검사한다.
- 메시지를 출력한다.
  - 반환받은 메시지를 출력한다.
  - 사용자가 잘못된 값[1-9]를 벗어난 값을 입력할 경우, [ERROR] 메시지 처리한다.
