# 숫자 야구 게임
## 진행 방법
* 숫자 야구 게임 요구사항을 파악한다.
* 요구사항에 대한 구현을 완료한 후 자신의 github 아이디에 해당하는 브랜치에 Pull Request(이하 PR)를 통해 과제를 제출한다.

## 과제 제출 과정
* [과제 제출 방법](https://github.com/next-step/nextstep-docs/tree/master/precourse)

## 기능 목록
1. 읽은 문자열의 유효성을 검증한다. / Controller , Util
    1. 길이가 3개가 맞는지
    2. 오직 숫자만을 포함하고 있는지
    3. 중복된 숫자가 있는지
        1. 112, 122 등 숫자가 중복돼서는 안된다.

2. 컴퓨터의 공과 나의 공을 비교하여 스트라이크, 볼, 낫싱을 반환한다. / Model
    1. 과정을 세분화한다.
        1. 1 : 1 로 비교하는 과정 (공 한개와 공 한개를 비교한다)
        2. 1 : 3 으로 비교하는 과정 (공 세개와 공 한개를 비교한다)
        3. 3 : 3 으로 비교하는 과정 (공 세개와 공 세개를 비교한다)

3. 랜덤으로 공의 번호를 생성한다. / Controller
    1. 중복되는 숫자가 있는지 체크한다 (위의 유효성 검증로직을 이용할 수 있다)
        1. 중복되는 숫자가 있다면 재생성한다
        2. 중복되는 숫자가 없다면 종료한다
    2. 정확히 3글자인지 체크한다. (위의 유효성 검증로직을 이용할 수 있다)

4. 출력 및 입력을 받는 객체를 생성한다. / View
    1. 입출력과정
        1. '숫자를 입력해주세요' 를 출력한다
        2. 입력을 받는다
        3. 결과를 출력한다
           1. 잘못된 입력인 경우 [ERROR] 로 시작하는 에러메시지를 출력하도록 한다.
    2. 게임이 끝난 경우
        1. 재시작 할 것인지
        2. 종료할 것인지 에 대한 입력을 받은 뒤, 처리한다
