# 숫자 야구 게임

## 진행 방법

* 숫자 야구 게임 요구사항을 파악한다.
* 요구사항에 대한 구현을 완료한 후 자신의 github 아이디에 해당하는 브랜치에 Pull Request(이하 PR)를 통해 과제를 제출한다.

## 과제 제출 과정

* [과제 제출 방법](https://github.com/next-step/nextstep-docs/tree/master/precourse)

## 기능 요구사항

`상대방`이 정한 1부터 9까지 서로 다른 수로 이루어진 3자리 수를 `사용자`가 맞추는 게임

1. 게임의 운영
    1. '사용자'는 애플리케이션을 실행하며 '상대방'은 컴퓨터로 한다.
    1. 애플리케이션이 실행되면 아래 메세지가 출력되며 사용자의 입력을 받는다. 사용자가 값을 입력하면 룰에 따라 상대방의 결과와 비교하여 그 결과를 출력한다.

        ```
        숫자를 입력해주세요 : {사용자 입력 위치}
       {결과값 메세지 위치}
        ```
    1. 룰에 따라 게임이 종료되면 새로운 게임을 시작하거나 완전히 종료할 수 있다.
        1. 게임이 종료되면 아래 메세지가 출력되며 사용자의 입력을 받는다.

            ```
            게임을 새로 시작하려면 1, 종료하려면 2를 입력하세요.
            {사용자 입력 위치}
            ```
    1. 사용자가 잘못된 값을 입력하는 경우 아래와 같은 형태의 메세지를 출력하고 진행중이던 게임을 계속 진행한다.

        ```
        [ERROR] {메세지 내용}
        ```
1. 룰
    1. '정답'이란 상대방이 생성하고 사용자가 맞춰야하는 값을 의미한다. '정답'은 1~9까지 서로 다른수로 이루어진 3자리 수이다.
    1. 게임을 시작하면서 상대방은 정답을 정한다.
    1. 사용자가 정답을 입력하면, 상대방의 정답과 비교하여 그 결과를 출력한다. 그 예는 아래와 같다.
        * 1스트라이크 1볼
        * 3볼
        * 2스트라이크
        * 낫싱
    1. 숫자, 위치가 일치하면 `스트라이크`이다.
    1. 숫자가 일치하고 위치가 일치하지 않으면  `볼`이다.
    1. `스트라이크` 또는 `볼`이 없으면 `낫싱`이다.
    1. 숫자를 모두 맞히면 게임이 종료되며 그 이후는 운영 방식에 따른다.

## 기능 목록

* 게임 컨트롤러 : 게임의 시작, 반복, 종료를 관리
* 입력 : 사용자의 입력을 관리
* 출력 : 게임의 진행, 결과를 출력
* 게임 모델
    * 정답 검증 : 사용자의 입력값을 룰에 따라 검증
    * 유효성 검증 : 사용자의 입력값이 유효한 입력인지 검증
    * 정답 생성 : 게임의 정답을 생성
