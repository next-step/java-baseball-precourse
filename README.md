# 숫자 야구 게임
## 진행 방법
* 숫자 야구 게임 요구사항을 파악한다.
* 요구사항에 대한 구현을 완료한 후 자신의 github 아이디에 해당하는 브랜치에 Pull Request(이하 PR)를 통해 과제를 제출한다.

## 과제 제출 과정
* [과제 제출 방법](https://github.com/next-step/nextstep-docs/tree/master/precourse)

## 야구 게임 요구사항
- [x] 공 (Ball)
    - [X] 공은 위치 정보와 숫자 정보를 가지고 있다.
    - [x] 공의 숫자는 1부터 9사이의 값을 가져야 한다.
    - [x] 공의 위치의 값은 1이상 3이하 이여야 한다.
    - [x] 공의 위치 정보와 숫자가 입력 가능한 범위를 벗어날 경우 에러를 발생한다.


- [x] 세자리의 공 (Balls)
    - [x] 세자리의 공을 가져야 한다.  
    - [x] 사용자가 중복되는 공 입력 시 에러를 발생한다.


- [x] 심판 (Referee)
    - [x] 컴퓨터의 세자리의 공과 사용자의 공 하나를 비교하여 위치정보와 공의 숫자가 일치하면 스트라이크(STRIKE)을 반환한다.
    - [x] 컴퓨터의 세자리의 공과 사용자의 공 하나를 비교하여 위치정보는 다르나 공의 숫자가 일치하면 볼(BALL)을 반환한다.
    - [x] 컴퓨터의 세자리의 공과 사용자의 공 하나를 비교하여 공의 숫자가 일치하는것이 없다면 낫씽(NOTHING)을 반환한다.


- [x] 결과 (GameResult)
    - [x] 컴퓨터 세자리의 공과 사용자의 세자리의 공을 비교하여 나온 결과값의 전체를 반환한다.
    - [ ] 3스트라이크 일 경우 게임을 종료하거나 다시 시작할 수 있다.

- [x] UI
    - [x] 컴퓨터의 세자리 공은 임의의 수 3개가 된다.
    - [x] 컴퓨터가 선택한 3개의 숫자를 모두 맞히면 게임 종료
