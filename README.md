## 🚀 기능 요구사항
- 이 게임은 프로그램이 1에서 9까지 서로 다른 임의의 수 3개를 정하고 이를 플레이어가 맞추는 게임이다.
- 정답을 맞추기 위해 3자리수를 입력하고 힌트를 받는다.
- 힌트는 야구용어인 볼과 스트라이크로 받는데, 같은 자리에 같은 숫자가 있는 경우를 `스트라이크`, 다른 자리에 숫자가 있는 경우를 `볼`로 정한다.
- 같은 숫자가 하나도 없는 경우 힌트로 `낫싱`을 받는다.
    - 예시) 상대방(컴퓨터)의 수가 425일 때, 123을 제시한 경우: 1스트라이크, 456을 제시한 경우: 1볼 1스트라이크, 789를 제시한 경우: 낫싱
- 3자리 숫자가 정답과 같은 경우 게임이 종료된다.
- 게임을 종료한 후 게임을 다시 시작하거나 완전히 종료할 수 있다.
- 아래의 프로그래밍 실행 결과 예시와 동일하게 입력과 출력이 이루어져야 한다.

## 구현 기능 목록

### 랜덤 값 생성(Computer)
- 각 자리는 1~9까지의 수로 이루어져 있다.
- 각 자리는 서로 다른 수로 이루어져 있다.
- 3자리 수보다 적거나 많은 자리의 수는 허용되지 않는다.
- 랜덤 값은 List로 반환한다.(get)

### 사용자 입력 (Player)
- 각 자리는 1~9까지의 수로 입력받아야 한다.
- 각 자리는 서로 다른 수로 입력받아야 한다.
- 3자리 수보다 적거나 많은 자리의 수는 허용되지 않는다.
- 사용자 입력 값은 List로 변환하여 반환한다. (get)

### 판단 (Judgement)
- Player가 입력한 수와 Computer가 생성한 수를 비교하여 Judgement를 반환한다.
- 판단은 게임에서 승리했는지 여부가 반환된다.
- 판단 후 판단 결과를 출력한다.(스트라이크 count 와 Ball count)
    - 스트라이크 : 3자리의 랜덤 수와 사용자가 입력한 3자리의 수 중 위치와 수가 모두 같은 경우를 말한다.
    - 볼 : 3자리의 랜덤 수와 사용자가 입력한 3자리의 수 중 위치는 다르지만 수가 같은 경우
    - 낫싱 : 3자리의 랜덤 수와 사용자가 입력한 3자리의 수가 모두 다른 경우
- 승리 (Victory) 시 게임 재시작, 종료와 관련한 의견(Opinion)을 입력할 수 있다.
- 승리하지 않은 경우 게임을 계속 진행한다.

### 게임 실행 (GameRunner)
- 숫자 야구 게임 맞추기를 실행한다.
- Computer 및 Player 생성하여 판단 후 결과를 출력한다.

### 의견 (Opinion)
- 게임 승리 후 게임을 계속할 것인지, 종료할 것인지 선택한다.
- 재시작 (1), 종료(2)

### 게임룰 (GameRule)
- 1~9까지의 수
- 3자리수
- 재시작(RESET) 1
- 종료(QUIT) 2
- 위와 같은 상수를 정의하여 가져다쓴다.

### 출력 (OutputView)
- 출력 스트링 및 에러 문자를 정의하여 가져다쓴다.
