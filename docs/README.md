### 요구사항 정리
<pre>
- 숫자범위:1~9, 서로 다른 수, 3자리
- 프로그램(컴퓨터)은 위의 숫자 조합으로 Random하게 숫자를 생성
    - 요구사항에 따라 pickNumberInRange() 활용
    - 기본 제공 테스트코드를 위해 pickUniqueNumbersInRange() 미활용

- 사용자는 Console을 통해 숫자 3자리를 입력

- 사용자가 잘못된 값을 입력 시 예외발생
    - 입력한 값이 잘못된 길이인 경우
    - 숫자가 아니거나 0인 경우
    - 서로다른 수가 아닌 경우

- 프로그램이 생성한 숫자와 사용자가 입력한 숫자를 비교
    - 위치와 숫자가 모두 동일한 경우 스트라이크
    - 위치는 다르지만 동일한 숫자가 있는 경우 볼
    - 위치와 숫자가 모두 다른 경우 낫씽

- 모두 맞추는 경우 종료
- 게임 종료 후 특정 숫자 입력 시 재수행 or 프로그램 종료
</pre>

### 요구사항 기반으로 테스트코드 작성(TDD)
<pre>
Computer(컴퓨터, 프로그램)
- 생성된 난수의 길이가 3인지 확인
- mocking 하여 지정된 난수 생성 시 이슈 발생 확인

User(사용자, 게임참가자)
- 입력한 숫자의 길이가 3인지 확인
- 입력한 숫자 파싱하여 numbers(List)에 정상 저장여부 확인

Validation(사용자 입력 문자열 검증)
- 입력받은 문자열 길이 확인(default: 3)
- 입력받은 문자열 숫자형식, 혹은 0이 아닌지 확인
- 서로다른 수가 맞는지 체크

Game(사용자와 컴퓨터의 게임결과)
- mocking 하여 컴퓨터의 난수 생성 값 세팅 후
  사용자의 생성 값을 세팅 후 예상되는 결과 값 확인
</pre>
